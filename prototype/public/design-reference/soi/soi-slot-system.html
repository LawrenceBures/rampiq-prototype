Architecture

SOI is built on a fixed grid of anchors and slots. Anchors never move. Slots accept modules via snap-only placement. There is no freeform dragging — every layout is composed, never improvised.

Anchors · Permanent

A1 · Header. Operator identity, station, time, conditions, SOI status.
M1 · Map. Ambient pressure field. Visualization can switch, position is fixed.
A2 · Active Recommendation. The decision surface. Expands in Crisis Mode.
A3 · Command Dock. AI presence and voice/text input. Always at bottom.

Slots · Customizable

L1–L4 · Left rail. 4 vertical slots.
R1–R4 · Right rail. 4 vertical slots.
U1–U2 · Lower utility. 2 horizontal slots under the recommendation panel.

Slot Sizes

Compact
Header + 1-line summary. Used for de-emphasized modules.
Normal
Default size. Full content visible. Most modules ship here.
Expanded
Takes 2 slot positions. Used for emphasis or richer detail.

Placement Rules

Modules snap into slots. No freeform pixel positioning.
A module can only sit in one slot at a time.
An Expanded module consumes 2 adjacent slots in the same rail.
Empty slots are valid and visible only in Edit Mode.
Anchors cannot be moved, hidden, resized, or replaced.
